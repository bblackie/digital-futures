## Key Concepts (What You Must Understand)

- **Variable:** A named container that stores a value
- **Data type:** The kind of value (number, string, true/false, array, etc.)
- **Operator:** A symbol that does something (+, >, &&, etc.)
- **Conditional:** if/else—code that runs only if a condition is true
- **Loop:** Code that repeats (for, while)
- **Function:** A reusable block of code that does one job
- **Scope:** Where a variable can be used (global = everywhere, local = only in function)
- **DOM:** The structure of an HTML page that JavaScript can interact with
- **Event:** User action (click, type, submit) that triggers code

If you can't explain conditionals, loops, or functions, reread this section.

## Big Ideas

### 1. Programming Is Problem-Solving, Not Just Coding

A good programmer:
- Breaks the problem into small steps
- Writes one step at a time and tests it
- Reads error messages and learns from them
- Iterates until it works

Writing code is the easy part. *Thinking* is the hard part.

### 2. You Won't Remember Everything

JavaScript has thousands of functions and methods. You don't memorize them. You:
- Understand the core concepts (variables, loops, functions)
- Look up specific syntax as needed (Google, MDN, ChatGPT)
- Practice until common patterns become automatic

### 3. Debugging Is Part of the Job

Your code *will* have errors. That's normal. Professional programmers spend 30–40% of their time debugging. The skill is reading error messages and knowing where to look.

## Common Pitfalls

🚩 **"I don't understand why my code isn't working."**  
Add `console.log()` statements to print variable values. This shows you what's happening at each step.

🚩 **"My code is 200 lines long and hard to read."**  
Break it into functions. Each function should do one job and fit on one screen. This is refactoring.

🚩 **"I copied code from the internet but I don't understand it."**  
You must understand every line. Read it, type it yourself (don't copy-paste), modify it, and test it.

🚩 **"The error message says 'undefined is not a function.' I don't know what this means."**  
This usually means you're trying to call a variable as a function (missing parentheses), or a variable doesn't exist. Check spelling.

## Big Vocabulary List

- **Variable:** Named container for data
- **Const/Let/Var:** Keywords to declare variables (const = recommended, var = old, let = flexible)
- **String:** Text data (e.g., "hello", 'world')
- **Number:** Numeric data (e.g., 5, 3.14, -10)
- **Boolean:** True or false
- **Array:** List of values (e.g., [1, 2, 3, "apple"])
- **Object:** Collection of key-value pairs (e.g., `{name: "Alice", age: 15}`)
- **Operator:** Symbol that does something (+, -, >, &&, ||, etc.)
- **Conditional:** if/else statement
- **Loop:** for or while loop
- **Function:** Reusable block of code
- **Parameter:** Input to a function
- **Return:** Output from a function
- **Scope:** Where a variable is accessible (global or local)
- **DOM:** Document Object Model (the structure of an HTML page)
- **Console:** Browser tool for running JavaScript and seeing output
- **Debug:** Find and fix errors in code
