## 1. Purpose of These Notes

A professional 3D artist's Blender file looks organized. Files are named clearly, objects are grouped in collections, versions are saved regularly, and assets are stored separately.

This topic teaches you:
- How to name things consistently
- How to use collections (folders) to organize your scene
- How to save versions (so you can rollback if something breaks)
- How to export your model for use in other projects
- How to back up your work

---

## 2. Key Concepts

- **Collections:** Folders within a Blender file that organize objects logically
- **Naming conventions:** Consistent patterns for object names (e.g., `Body_MainMesh`, `Eyes_Left`, `Rig_Armature`)
- **Version control:** Saving multiple versions of your file (v1, v2, v3, etc.) so you can revert if needed
- **File organization:** Where you store Blender files, textures, and exported models on your computer
- **Export formats:** Choosing the right file format for your model (FBX, OBJ, GLTF) depending on where it will be used

If you can't organize a scene into collections or explain why version control matters, reread this section.

## 3. Core Explanation

### Naming Conventions

**Bad names:**
- Cube
- Cube.001
- Mesh
- Object1

**Good names:**
- Body_MainMesh
- Eyes_Left
- Eyes_Right
- Eyelids_Open
- Rig_Armature
- Light_Key
- Light_Fill
- Light_Back

**Rules:**
1. Use descriptive words that say *what* the object is
2. Separate words with underscores (not spaces)
3. Use suffixes to denote type (e.g., `_Rig` for a rig, `_Light` for lights)
4. Use left/right suffixes for paired objects (e.g., `Arm_Left`, `Arm_Right`)
5. Avoid numbers unless necessary (numbers are for versions, not object identity)

**Why it matters:**
- Later, when you have 50 objects in your scene, you need to find "Arm_Left" quickly
- If you share your file with a classmate or teacher, they understand what each object is
- Versioning becomes easier when you're tracking meaningful file names

**How to create a collection:**
1. In the outliner (top-right), right-click в†’ "New Collection"
2. Name it (e.g., "Geometry", "Lights", "Rigs")
3. Drag objects into the collection by dragging them in the outliner

**Benefits:**
- Easy to hide/show groups of objects (click the eye icon next to a collection)
- Scene stays visually organized
- Easy to move objects between scenes without losing track

### Version Control: Saving Multiple Versions

**Bad practice:**
- You model for 2 hours, save once at the end as `chair.blend`
- You make a mistake and undo 50 steps
- Your work is lost

**Good practice:**
- Save regularly with version numbers: `chair_v1.blend`, `chair_v2.blend`, `chair_v3.blend`
- Each version is a checkpoint
- If you break something, you can always open a previous version and start from there

**How to do it:**
1. After a major milestone (finishing the basic shape, finishing detail, finishing textures), press `File в†’ Save As`
2. Name it with a version number: `modelname_v1`, `modelname_v2`, etc.
3. Continue working
4. When you've made significant progress, save a new version

**How many versions?** 3вЂ“5 per project is typical. Don't save every 2 minutes (you'll have 100 versions), but do save before major changes.

### File Organization on Your Computer

**Bad structure:**
```

  chair2.blend
  blender files/
    model1.blend
    texture.jpg

```

**Good structure:**
```

  Chair_Project/
    Blender_Files/
      chair_v1.blend
      chair_v2.blend
      chair_v3.blend
    Textures/
      wood.jpg
      fabric.jpg
    Renders/
      render_front.png
      render_side.png
    Exports/
      chair_final.fbx
      chair_final.obj

      chair_reference_01.jpg
      chair_reference_02.jpg
```

**Benefits:**
- Everything related to the project is in one folder
- Easy to find assets (textures, references)
- Easy to export and share
- Backups are simple (copy the entire folder)

### Linking Textures and External Files

If you use external texture files (images), link them to your Blender file so they load automatically.

**Bad practice:**
- Save a texture to your desktop
- In Blender, load the image from your desktop
- Move the desktop file and the texture disappears from your model

**Good practice:**
1. Create a `Textures/` folder inside your project folder
2. Keep all texture files there
3. In Blender, load textures from the project folder (relative path)
4. If you move the project folder, all textures move with it

### Exporting Your Model

When you're done, you'll export your model so it can be used elsewhere (in a game, animation, or other project).

**Export options:**

| Format | Best For | Notes |
|--------|----------|-------|
| **FBX** | Game engines, animation | Universal format, supports rigs and animations |
| **OBJ** | Static models, 3D printing | Simple, widely supported, no materials |
| **GLTF / GLB** | Web, games, real-time apps | Modern, efficient, includes materials |
| **STL** | 3D printing | Designed for physical models |

**How to export:**
1. File в†’ Export As в†’ Choose format
2. Name your file (e.g., `chair_final.fbx`)
3. Choose location (the `Exports/` folder)
4. Click the export button
5. Configure settings if needed (usually defaults are fine)

**Pro tip:** Always export to a separate folder, not your working Blender file folder. This keeps things clean.

### Backing Up Your Work

Accidents happen. Your computer crashes, a file corrupts, or you accidentally overwrite something. Backups save you.

**Simple backup method:**
1. Every few days, copy your entire project folder to:
   - An external drive, OR
   - A cloud service (Google Drive, OneDrive, Dropbox), OR
   - Both

**Advanced backup (cloud-based):**
- Use Git/GitHub (advanced, but professional)
- Save your Blender files to a Git repository
- This gives you version history + off-site backup

For Year 10, a simple copy-to-cloud-once-per-week backup is fine.

## 5. Worked Example: Organizing a Character Model

**Scenario:** You've modeled a character with a body, head, two arms, two legs, and a rig. You've also created 3 lights and a camera.

**Current mess:** 12 objects at the top level, no organization.

**Steps to organize:**

1. **Create collections:**

## 6. Common Misconceptions and Pitfalls

### вќЊ "I have 100 versions of my file. My folder is a mess."

This usually means you're saving too frequently. Save a new version only after a *major* milestone (finished geometry, finished texturing, ready for export). 3вЂ“5 versions per project is normal.

### вќЊ "I deleted an object and don't know its name anymore."

Press `Ctrl+Z` (undo) to get it back. If you undo too far and lose recent work, open the previous version file.

### вќЊ "My texture disappeared when I moved the project folder."

The texture was linked with an *absolute path* (C:\Users\...\texture.jpg), which broke when you moved the folder. Next time, use *relative paths* (load textures from a Textures/ subfolder within your project).

### вќЊ "I exported the model but it's invisible in the game / animation software."

This usually means:
- Geometry is there but it's off-screen (scale is wrong)
- Materials didn't export (export as FBX or GLTF, not OBJ, for materials)
- Model is rotated or scaled unexpectedly

Check the export settings and test with a simple model first.

### вќЊ "My computer crashed and I lost 2 hours of work."

That's why you save versions. If you only had one file, it's gone. Next time, save multiple versions and back up to a cloud drive once per day.

## 9. Key Vocabulary

- **Collection:** A folder within a Blender file that organizes objects
- **Outliner:** The hierarchy view in Blender that shows all objects and collections
- **Naming convention:** A consistent pattern for naming objects (e.g., Type_Description_Side)
- **Version control:** Saving multiple versions of a file so you can revert if needed
- **Relative path:** File path relative to the project folder (e.g., `Textures/wood.jpg`)
- **Absolute path:** Full file path from the root drive (e.g., `C:\Users\...\wood.jpg`)
- **Export:** Save a 3D model in a different format (FBX, OBJ, etc.) for use in other software
- **FBX:** Universal 3D file format that includes geometry, materials, and rigging
- **OBJ:** Simple 3D format for static geometry only
- **GLTF / GLB:** Modern, efficient format for web and games
- **Backup:** Copy of your files saved in a separate location (external drive, cloud storage)
