## What You'll Learn

This unit introduces **spatial thinking** and **3D digital creation** using Blender. You'll:

1. Understand how 3D space works (coordinates, axes, perspective)
2. Navigate and manipulate 3D environments
3. Create 3D objects using core modelling techniques
4. Apply materials and textures to make objects visually interesting
5. Light and render your models
6. Organize your work professionally using layers and naming conventions

By the end of this unit, you'll be able to:
- ✅ model a 3D object from a reference image or brief
- ✅ texture and light it realistically
- ✅ export it for use in other projects (games, animations, renderings)
- ✅ explain your modelling decisions and process

## How to Approach This Unit

**In class:**
- Follow tutorials step-by-step in your own Blender file
- Ask questions when something doesn't make sense
- Save multiple versions (model_v1, model_v2, model_v3) so you can revert if you break something
- Show your teacher your progress regularly (rough block-out, textured version, lit render)

**In lab work:**
- Start with simple shapes (boxes, spheres) before complex models
- Reference real objects or reference images—don't guess proportions
- Test your UV map and materials early, not at the last minute
- Render frequently to see how lighting affects your model

**When stuck:**
- Check the "Common Misconceptions" section in the relevant topic
- Rewatch the tutorial or try a different reference
- Ask a peer or your teacher
- Don't give up on a model—push through by simplifying it

## Big Ideas

### 1. You Can't Break Your Work Permanently
If you make a mistake:
- Press Ctrl+Z (undo) to go back
- Save versions (File → Save As) so you always have a backup
- Delete geometry and start over on a single feature
- This is why we iterate, not worry

### 2. Reference Images Are Your Friend
Every professional 3D artist uses reference images. Find:
- Front, side, and top views of what you're modelling
- Photos of real objects from similar angles
- Concept art or sketches
- Then model to match—don't guess

### 3. Lighting Sells the Model
A poorly lit, high-quality model looks bad. A simple model with great lighting looks impressive. Spend time on lighting.

### 4. Name Things Consistently
"Cube.001" is not a name. Use names like:
- `Body_MainMesh`
- `Eyes_Left`
- `Handle_Wood_Material`

This makes organization easy and file collaboration possible.

## Common Pitfalls in This Unit

🚩 **"I started texturing before I finished the model geometry."**  
This wastes time. Finish geometry first, texture later.

🚩 **"I can't find the extrude tool."**  
It's the `E` key. Use keyboard shortcuts—they're faster.

🚩 **"My model looks flat and uninteresting."**  
This is usually a lighting problem, not a modelling problem. Add more lights and try different angles.

🚩 **"I don't have a reference image."**  
Go find one on Google Images, reference.pictures, or Concept Art World. Don't guess.

🚩 **"I modelled in sculpture mode instead of box modelling."**  
Box modelling (extrude, scale, loop cut) is the standard workflow for hard-surface objects. Sculpture mode (Sculpting workspace) is for organic shapes only. Know which is which.

## Key Vocabulary

- **Mesh:** A 3D object made of vertices (points), edges (lines connecting points), and faces (surfaces between edges)
- **Vertex (plural: vertices):** A single point in 3D space; the fundamental building block of a mesh
- **Edge:** A line connecting two vertices
- **Face:** A surface area bounded by three or more edges; what you see on the outside of a model
- **Extrude:** The action of pulling a face outward to add geometry; your main modelling tool
- **Modifier:** A non-destructive effect applied to a mesh (e.g., Subdivision Surface makes a model smoother)
- **Topology:** The flow and arrangement of edge loops in a mesh; good topology makes animation and deformation easier
- **UV map / UV unwrapping:** The process of flattening a 3D surface so a 2D texture can be applied to it
- **Texture:** A 2D image applied to a 3D surface (e.g., wood grain, brick, fabric)
- **Material:** The collection of properties that define how a surface looks and behaves (colour, roughness, metallic, etc.)
- **Shader:** A program that defines how a surface reacts to light; the Principled BSDF is the standard shader
- **Normal map:** A special texture that makes a surface look like it has detail without adding geometry
- **Three-point lighting:** A professional lighting setup using key light (main), fill light (shadow softener), and back light (separation)
- **Render:** The process of calculating how light bounces off surfaces to create a final image
- **HDRI:** High Dynamic Range Image; a spherical image used for realistic environmental lighting
- **Version control:** Saving multiple versions of a file (model_v1, model_v2) to track progress and allow rollback
