# Unit 3: Game Design with GDevelop – Unit Overview

**Course:** 10DGTA – Digital Technologies Year 10  
**Unit:** Game Design  
**Duration:** Term 1 (~2 weeks)  
**Tool:** GDevelop 5 (free, event-based game engine)  
**Aligned Learning Outcomes:** Designing & Developing Digital Outcomes (Progress Outcome 3); Computational Thinking (Progress Outcome 4)—students design and develop digital content through iterative systems thinking

---

## What You'll Learn

This unit introduces **game design thinking** and **how to build playable games** without traditional coding. You'll:

1. Understand game mechanics (rules, feedback loops, win/loss conditions)
2. Use GDevelop's event system to create interactivity
3. Design a game brief and scope it realistically
4. Playtest with peers and iterate based on feedback
5. Understand systems thinking (how different game elements interact)

By the end of this unit, you'll be able to:
- ✅ design and build a simple playable game (arcade style, 2D)
- ✅ describe the mechanics and rules of your game clearly
- ✅ playtest with others and gather meaningful feedback
- ✅ iterate your game based on testing data

## Key Concepts (What You Must Understand)

- **Mechanic:** A rule or system in a game (jump, collect, defeat enemy)
- **Feedback loop:** How the game responds to player actions (visual, audio, score)
- **Win/loss condition:** How a player wins or loses
- **Event (in GDevelop):** A rule: "If [condition], then [action]"
- **Collision detection:** When two objects touch, something happens
- **Difficulty curve:** How a game gradually becomes harder to stay engaging
- **Playtest:** Testing with real players to find problems and opportunities
- **Scope:** What's realistic to finish in the time available

If you can't explain what a mechanic is or describe how a feedback loop works, reread this section.

## Common Pitfalls

🚩 **"I made a game but it's no fun to play."**  
This is a design/balance issue. Playtest with peers and ask: "What's boring? What's frustrating?" Then iterate.

🚩 **"My game is too hard / too easy."**  
Test with peers to calibrate difficulty. Adjust enemy speed, player damage, lives, or level layout based on feedback.

🚩 **"I spent all my time on graphics. Now I have no time for mechanics."**  
Focus on fun first. Graphics second. A simple game that's fun beats a beautiful game that's boring.

🚩 **"The game is buggy. Events aren't working."**  
Check event order (events execute top-to-bottom). Add debug text to see variable values. Ask your teacher for help.
